## 二、各模組能力分析

### 2.1 UnifiedPredictor（統一預測器）

| 能力 | 狀態 | 說明 |
|------|------|------|
| 線性回歸 | ✓ | sklearn LinearRegression |
| XGBoost | ✓ | 需安裝 xgboost |
| LightGBM | ✓ | 需安裝 lightgbm |
| ONNX 包裝 | ✓ | 自動轉換 linear/xgb/lgb 為快速推論 |
| 多地平線輸出 | ✓ | 依 domain 的 horizons |
| 輸入格式 | ✓ | numpy, list, dict, pandas DataFrame |
| 資料正規化 | ✓ | StandardScaler 可選 |
| 模型儲存/載入 | ✓ | pickle |
| 評估指標 | ✓ | fit 回傳 train_rmse/mae/r2，evaluate() |
| LRU 快取 | ✓ | 64 條目 |
| 限流 | ✓ | Token Bucket 60 容量 |
| NaN/Inf 處理 | ✓ | 自動替換為 0 |

**效能基準（2000 樣本）**：
- 訓練時間：~2 ms
- 50 次預測：~3 ms
- predict_many 5000 樣本：~3 ms（約 15 萬 samples/s）

## 四、已知限制與風險

### 4.1 技術限制

| 項目 | 說明 |
|------|------|
| 限流 | predict 每分鐘約 60 次，超過會 RuntimeError |
| 多資料源時間對齊 | 僅按 timestamp 排序，無交叉對齊 |
| canonical 對齊 | 缺失特徵填 0，可能影響泛化 |
| JSONL 儲存 | 大資料量時 I/O 較慢 |
| OWID | 需穩定網路，離線無法爬取 |

### 4.2 測試缺口

- `test_data_connectors.py`：匯入 YahooFinanceConnector 等失敗（__init__.py 未匯出）
- `test_performance.py`：使用舊 API（model=, task=, target=）
- ONNX 整合、CLI 整合測試未驗證

### 4.3 依賴風險

- yfinance：Yahoo 變更時可能失效
- NewsAPI、EIA：需 API key 才有真實資料
- scikit-learn：必要，無時退回 dict 模型
