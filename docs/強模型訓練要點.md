# 如何把預測模型訓練得更強

演算法只是上限的一部分；**資料量、特徵、標籤定義**才是決定「有多強」的主因。

## 1. 本專案已提供的強化手段

| 手段 | 說明 | 指令／設定 |
|------|------|------------|
| **衍生時序特徵** | 對 OHLCV 等欄位加滯後、滾動均值／標準差，讓樹模型能學「慣性」 | `python crawler_train_pipeline.py TASK --rich-features` 或 YAML `feature_expansion.enabled: true` |
| **強預設（樹更深、更多棵樹）** | 較低 learning_rate、較多 estimator | `--preset strong` |
| **Early stopping** | 依時間切後段當驗證集，減少過擬合（樣本夠時自動啟用） | 內建於 `fit()`，可用 `early_stopping=False` 關閉 |
| **Ensemble** | 線性迴歸 + XGB 加權平均，常比單一線性穩 | `--model ensemble`（需 `pip install xgboost`） |
| **AutoML（Optuna）** | 自動在 linear / xgboost / lightgbm（及可選 mlp、LSTM、Transformer）間搜超參 | `pip install -r requirements-automl.txt` 後 `--model automl --automl-trials 40`；深度請加 `--automl-include-deep` |
| **LSTM / Transformer** | 時序滑動視窗深度模型（樣本要夠） | `--model lstm` 或 `--model transformer`（需 torch）；walk-forward 預設跳過深度模型，需加 **`--wf-allow-deep`** |
| **CLI 超參** | 手動調 Torch 不必改程式 | `--seq-len`、`--torch-hidden`、`--torch-epochs`、`--torch-lr`、`--torch-batch-size`、`--lstm-num-layers`、`--transformer-nhead` 等 |
| **實驗紀錄** | 區分「請求的 model」與 AutoML 實際選型 | `data/experiment_runs.jsonl` 內 `model_requested` / `model_trained` / `automl` |
| **近似區間** | 非共形，僅粗估 | `UnifiedPredictor.predict_interval_naive(X)`（以訓練 RMSE 當 σ） |
| **真實資料** | 離線後備只有約三十筆，上限很低 | `pip install yfinance`、拉長 `period: 1y`～`2y` |

## 2. 建議指令範例（盡力壓榨目前管線）

```bash
pip install xgboost yfinance pandas

python crawler_train_pipeline.py stock_price_next ^
  --model ensemble ^
  --preset strong ^
  --rich-features ^
  --period 1y ^
  --walk-forward
```

## 3. 仍無法靠「改參數」解決的事

- **深度模型上限**：仍強烈依賴資料量與特徵品質；小樣本時樹模型＋衍生特徵往往更穩。
- **超大資料、即時流**：需資料工程與基礎設施，非單檔腳本可替代。
- **因果／結構性預測**：需問題建模與外生變數，不是自動調 `n_estimators` 能達成。

結論：**先把資料變多、變真，再用 `--rich-features` + `xgboost`/`ensemble` + `--preset strong`**，才是這套程式裡最實際的「變強」路徑。
